## 1. 项目实现了什么功能 / 解决了什么问题

### 1.1 背景与痛点

**Vision Transformer（ViT）** 是当前最先进的图像理解模型之一。其核心机制——**自注意力（Self-Attention）**——决定了模型在分析一张图片时"把目光投向哪里"。然而，这套机制的内部运作对普通用户和初学者来说完全是一个"黑盒"：模型为何关注图像的某个区域，而忽略另一个区域，无法直观观察。

### 1.2 本项目的解决方案

**ViT Attention Visualizer** 是一个 **基于 Web 的交互式注意力可视化工具**。用户只需上传一张普通图片，系统即可自动：

1. 将图片输入 **DINOv3 ViT-S/16** 深度学习模型（Meta AI 发布的先进视觉预训练模型）
2. 通过**钩子（Hook）技术**，在模型推理过程中实时截获每一层、每一个注意力头的权重数据
3. 将这些抽象的数值数据转化为**彩色热力图**，叠加在原始图片上呈现

最终，用户可以看到：模型在看这张图时，12 层网络的 6 个注意力头（共 72 张热力图）**分别"关注"了图片的哪些区域**，如主体轮廓、纹理细节、背景等。

### 1.3 核心功能清单

| 功能模块 | 说明 |
|---|---|
| 图片上传 | 支持点击选择或拖拽上传，格式不限（JPEG、PNG 等） |
| 注意力提取 | 自动提取 12 层 × 6 头 = 72 张注意力权重图 |
| 3D 可视化 | 使用 Three.js 将注意力图以立体排列方式展示，支持旋转拖拽 |
| 2D 可视化 | 网格平铺展示所有注意力图，适合横向对比 |
| 热力图 / 灰度图切换 | 支持暖色 Viridis 热力图或灰度图两种配色模式 |
| 逐层筛选 | 滑动条可筛选某一层（Layer 1–12）或查看全部层 |
| 不透明度调节 | 实时调整注意力叠加层的透明度，便于与原图对比 |
| 完全本地运行 | 无需联网，所有推理在本机完成，数据不上传至第三方 |

- **演示汇报**：向非技术人员展示 AI 模型的"思考方式"，提升产品透明度与信任感。

## 3. 与同类项目相比的亮点

目前同类注意力可视化工具主要有两类：在线 Demo（如 Hugging Face Spaces）和 Jupyter Notebook 脚本。本项目相比之下具有以下差异化优势：

### 3.1 功能亮点对比

| 对比维度 | 典型同类工具 | 本项目 |
|---|---|---|
| 模型版本 | 旧版 ViT / DINO | **DINOv3 ViT-S/16**（含 RoPE、Register Tokens 等最新特性） |
| 可视化维度 | 仅 2D 平铺 | **3D 立体交互 + 2D 平铺**，可旋转拖拽 |
| 运行方式 | 依赖云端服务或在线 Hugging Face | **完全本地**，无需联网，数据安全 |
| 模型加载 | 运行时联网下载（受网络影响） | **本地 `.pth` 权重文件**，稳定可靠，零 Hub 缓存冲突 |
| 后端框架 | 无后端（纯前端）或 Flask | **FastAPI**，异步高性能，含规范 API 接口 |
| 注意力提取 | 修改模型源码 | **非侵入式 Forward Hook**，兼容原始模型结构 |
| 注意力范围 | 部分层/头 | **12 层 × 6 头 = 72 张**全覆盖 |

### 3.2 技术原创性亮点

- **非侵入式 Hook 方案**：DINOv3 原始代码使用 PyTorch 融合算子（`scaled_dot_product_attention`），该算子不暴露中间注意力矩阵。本项目通过 `register_forward_hook` 拦截输入，手动复现 QKV 分解、RoPE 应用与 Softmax 计算，在不修改模型结构的前提下提取完整注意力权重。这是本项目在工程实现上的核心创新点。
- **Register Token 感知**：正确跳过 DINOv3 特有的 4 个 Register Token（共 5 个 prefix token），确保提取的是纯图像 patch 的注意力，避免伪影干扰。
- **MIME 类型修复**：针对 Windows 环境下系统注册表错误导致 `.js` 模块加载失败的问题，进行了主动修复。

## 4. 如何运行与演示

> 以下说明面向需要本地运行项目的人员。

### 4.1 环境要求

- Python **3.11** 或更高版本
- 约 **2 GB** 磁盘空间（用于模型权重文件）
- 推荐使用 GPU，也支持纯 CPU 运行（速度较慢）

### 4.2 安装步骤

```bash
# 第一步：克隆代码仓库
git clone https://github.com/Y-chen3164553757/vit-attention-app.git
cd vit-attention-app

# 第二步：安装依赖库
pip install -r requirements.txt
```

### 4.3 准备模型权重

将 DINOv3 ViT-S/16 预训练权重文件（`dinov3_vits16_pretrain_lvd1689m-08c60483.pth`）
下载后放置于项目根目录下的 `weights/` 文件夹中：

```
vit-attention-app/
└── weights/
    └── dinov3_vits16_pretrain_lvd1689m-08c60483.pth  ← 放在这里
```

### 4.4 启动服务

```bash
python app.py
```

启动成功后，打开浏览器访问：

```
http://127.0.0.1:8080
```

### 4.5 演示操作流程

1. 页面加载完成后，默认展示一张预置的演示图片
2. 点击左侧**上传区域**或**拖拽图片**到该区域，选择任意图片
3. 系统自动运行模型推理（约 3–10 秒，取决于硬件）
4. 推理完成后，注意力热力图将以**3D 立体形式**呈现在屏幕中央
5. 可使用底部控制栏：
   - 切换 **3D / 2D** 视图
   - 切换 **热力图 / 灰度图** 配色
   - 拖动 **Layer 滑块** 查看特定层的注意力
   - 调整 **不透明度** 滑块控制热力图叠加强度
